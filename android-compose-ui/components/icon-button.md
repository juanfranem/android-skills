# Icon Button

## Use when

- The action is compact, repeated, or naturally icon-driven, such as search, share, favorite, or open menu.

## Variants

- `IconButton`
- `IconToggleButton`
- Filled, tonal, and outlined icon-button styles when stronger affordance is useful

## Guidance

- Use only when the icon meaning is clear.
- Add a meaningful `contentDescription` for interactive icon-only controls.
- Toggle variants fit binary state such as favorite/bookmark.

## Nearby alternatives

- Text plus icon or stronger emphasis -> button family
- Overflow action list -> menu triggered by icon button

## Common mistakes

- Icon without clear meaning.
- Missing semantics for assistive tech.
