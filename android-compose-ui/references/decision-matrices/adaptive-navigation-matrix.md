# Adaptive Navigation Matrix

- `NavigationBar`: best for compact windows with a small set of top-level destinations.
- `NavigationRail`: best for medium-width layouts where destinations stay top-level and persistent side navigation improves reachability.
- Navigation drawer: best when destination count, hierarchy, or account and context links exceed rail or bar density.
- Tabs: best for peer sections inside one feature, not app-wide navigation.
- Pane scaffolds: best when window conditions support simultaneous content regions and the information architecture benefits from side-by-side context; caveat when pane APIs are experimental or version-sensitive.
